# Subtask 3: FiT-QA (Figures and Tables Question Answering)

本研究で提案するタスクは、テキスト・図表・画像から構成される「栽培暦」のPDF文書を画像として入力し、与えられた質問に対して適切な回答を生成するマルチモーダル質問応答タスクです。

## 入出力データ・評価指標

### 入力データ

以下のファイルと質問（クエリ）を入力とします。

- 栽培暦（画像）: 栽培作物の栽培工程を地域の気候条件や作期に基づき時系列的に整理した農業技術資料
- 質問（クエリ）: 具体的な質問文

栽培暦の画像ファイルはこのリポジトリでは配布していませんが、同梱の `download.py` を実行することでダウンロード・生成できます（事前に `pip3 install tqdm PyMuPDF` を実行する必要があります）。

### 出力データ

- 回答の文字列 (回答は単語や短いフレーズにしてください。1文以上の長い形態や長い文章で記述することを禁止します。)

### 評価指標

- 正解の回答とシステムの回答との文字列類似度（BLEU）
- LLM-as-a-Judge

## データセット仕様

画像ファイル名のページ番号は、同梱の `download.py` が生成する PDF 内 0 始まりページ番号です。
たとえば PDF の先頭ページは `*_page_0.png` です。

### bboxについて

trainの各サンプルにはbbox(bounding box)という回答の根拠部分を示す矩形が付与されています。
回答の根拠部分を用いたVisual QAシステム開発の訓練・分析用にお使いください。

### difficulty について

trainの各サンプルには、質問応答の難しさを表す `difficulty` が付与されています。`difficulty` は回答文字列の長さではなく、主に質問に答えるために必要な推論の複雑さを表します。

- `easy`: 図表や本文中の単一箇所、またはごく局所的な範囲から答えを直接抽出できる質問で、LLMによる自動合成と人手によるpost-editで作成
- `difficult`: 複数箇所の情報の突き合わせ、比較、数え上げ、期間計算、条件付き判断などを要する質問で、完全に人手のみで作成

`difficulty` はデータ分析や難易度別評価のための補助情報として利用できます。

### 入力データ（fit-qa_train_dryrun_v3.jsonl）の例

```
{"question_id": "train_1", "file_name": "02-021-002_page_0.png", "question": "4月の「整枝剪定」で、小玉を少なくするために行う作業は何か。", "page": 0, "image_width": 1754, "image_height": 2481, "bbox": [1350.48, 254.75, 342.82, 61.55], "answer": "ハサミを使って着果部位を整える", "difficulty": "easy"}
```

### 入力データ（fit-qa_test_dryrun_v3.jsonl）の例

```
{"question_id": "test_7", "file_name": "02-021-001_page_0.png", "page": 0, "question": "2025年水稲栽培ごよみの作成機関はどこですか？"}
```

### 提出データの形式（jsonl形式）

```
{"question_id": "test_1", "answer": "..."}
{"question_id": "test_2", "answer": "..."}
...
```
