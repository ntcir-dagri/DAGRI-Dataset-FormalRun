# Subtask 2: Multi-Table Multi-Hop QA
Subtask 2 (Multi-Table Multi-Hop QA) では、経営指標、経営類型を構成する複数表を参照し、
農業経営に関する質問に回答可能なシステムの構築を目指します。

Subtask 2では、次のデータが配布されます。トップフォルダは自治体ごとになっています（自治体名は、`prefA`などと記載されています。）。
* 経営指標、経営類型を構成する表データ（JSON形式）
  * 経営指標: `input_{自治体名}/management_indicators/{品目名}`フォルダ以下のJSONです。
    * **balance.json**: 経営収支のデータです(各費目とjsonキーとの対応関係はSubtask 1を参照)。
    * **work_technologies.json**: 作業技術の一覧です。
      * "name": 作業名
      * "description": 簡単な作業の概要
      * "equipments" : 使用する機材（トラクターなど）とその使用量のリスト
      * "materials": 使用する資材とその使用料のリスト
      * "number_of_workers": この作業に必要な人員数
      * "total_number_of_hours": この作業にかかる延べ時間（人数分の総和）
      * "cost": この作業にかかる費用（未使用）
      * "note": 備考欄です。ここを参照する問題もあります。
    * **work_schedule**: 作業時期と作業時間の一覧です。
      * "name": 作業名（作業技術の一覧と表記が一致しないことがあります）
      * "period": 〇月×旬の時期が記載されています。
      * "hours": 作業時間
  * 経営類型: `input_{自治体名}/management_types/{類型名}`フォルダ以下のJSONです。
    * **balance.json**: 経営収支のデータです。費目は経営指標と同じです。
    * **premise.json**: 労働力、栽培する品目、この類型での所得目標などからなる前提表です。
      * "prefecture_name": 自治体名です。`自治体A`のように、抽象的な名前で記載されています。トップフォルダ名に含まれる`prefA`と同じ自治体を指しています。。
      * "area_name": 自治体内での特定地域です。
      * "crop_names": 栽培する品目名のリストです。品目名はmanagement_indicatorsフォルダ直下のフォルダ名と一致しています。
      * "cultivate_land"/"cultivate_land_unit": 類型の農家が使用するほ場（畑や水田）の総面積とその単位です。
      * "borrowed_cultivate_land"/"owned_cultivate_land": 上記の内、借地の面積と所持するものの面積です。
      * "labors"/"labor_raw": 家族労働力とその表記です。表記が数値とずれることはSubtask 2では存在しません。
      * "total_income": この類型で設定された所得額です。
      * "total_labor_hours": 家族労働力の合計値です。
      * "note": 備考欄です。
  * Subtask 1の出力データと同形式です。詳細はSubtask 1の提出データ形式およびサンプルデータをご確認ください。
* 質問応答サンプルデータ
  * 1問1答を1JSONオブジェクトとした、複数JSONからなるJSONL形式です。
  * 提出するデータも同様の形式としてください。


## サンプルデータおよび提出データ形式
````json
// sample.json
{"id": 0, "question": "人参の単価を答えよ。", "answer": "143円/kg"}
{"id": 2, "question": "大豆と春小麦で共通する作業機械名を列挙せよ。", "answer": ["マニュアスプレッダー", "フロントローダー","サブソイラー", "ブロードキャスター", "自走式汎用コンバイン", "ロータリーハロー", "鎮圧ローラー"]}
{"id": 114, "question": "人参を400a栽培する際,動力光熱費が2倍になったときに追加でいくらの動力光熱費が発生するか答えよ。また、この時の経営費を答えよ。", "answer": {"追加分": "241200円", "経営費": "503820円"}}
````

サンプルデータ、提出データは、次の要素からなるJSONオブジェクト列のJSONL形式です。
* id: 質問IDです。提供されるデータ（サンプル、提出データ両方）に値が記載されています。
* question: 質問文です。テキストとなっています。回答形式（JSONオブジェクトの構造について）の指定が記載されている場合もあります。サンプル、提出データ両方の値が記載されています。
* answer: 回答内容のJSONオブジェクトやリストです。サンプルデータには値が含まれています。提出データにはからの文字列が記載されたもの（`"answer":""`）が提供されます。空オブジェクトであっても、回答がリストである可能性があります（上記の`"id:2"`の例）


## 質問の種類
質問は、下記に示す種類の質問が設定されています。

1. 表中の特定の値を求める質問
2. 表中の複数の値を求める質問
3. 表中の複数の値による算出が必要な質問
4. 表中の値の操作（列ずらしなど）を行う質問
5. 簿記・農業簿記などに基づく知識を必要とする質問（限界利益の導出など）
5. 1～5について、複数表から回答を求める質問
  1. 複数品目についての同種の表から導出する質問（異なる品目での労働時間比較の質問など）
  2. 同一品目の異なる表から導出する質問（栽培技術に必要な機材と作業時期の質問など）
  3. 複数品目、複数種の表から導出する質問（上記すべての組み合わせ）


### 出題意図

経営指標を読解できるシステムは、農業経営における構成要素を理解できると期待できます。
特に、質問種類の4や5などの類型を調整する処理に対応できるシステムは、農業経営のモデリングにたいして強いシステムであると期待できます。
また、経営指標として作成されているデータは、都道府県で類似した項目が設定されています。
このことから、様々な地域に対応できるシステムを構築することができれば、日本国内での地域特性を理解した経営シミュレーションが可能となると期待できます。

本タスクでは、複数表から構成される経営指標をマルチテーブルを対象としたマルチホップQAと捉えることで、
農業分野に適した経営シミュレーション


## 評価方法
Subtask 2は、質問に対し正確な回答ができているかで評価します。
正確性は**正解との完全一致**した回答数の割合で評価します。

完全一致で評価するため、システムの出力する回答の形式を下記の通りに定めます。
* 英数記号は半角、それ以外は全角。
  * 大文字小文字は配布データに記載の通り
* 表から1つの値の抽出を行う質問:
  * 数値を求める質問の場合は、数値+単位を回答（例: 143円、0.2h）
    * 金額を求める場合は、単位は**円**
    * 時間を求める場合は、単位は**h**
    * 何か月かを求める場合は、単位は**ヶ月**
    * 人数は、数量を半角数字で記載し、**人**
    * それ以外の単位は、配布データに記載の通り
    * 面積あたりについては、10a面積当たりのみ（/5aや/1haなどの回答を要求することはない）
　* 時期を求める場合:
    * 何月かを求める場合（例: 収穫作業を行う月を答えよ --> "4月"）
    * 月の上中下旬のいずれかを求める場合（例: 春小麦の播種時期を答えよ --> "4月中旬" ）
* 1質問に対し複数回答が必要な質問:
  * 列挙が必要な質問は、リストで回答
    * 例1: **質問:** *緑肥の品種名の列挙* --> **回答:** *["ヘイオーツ", "キラカシ", "ヘアリーベッチ"]*
    * 例2: **質問** *人参の耕起作業における作業時期と作業時間* --> **回答** *["4月下旬", "0.1h/10a"]*
    * 構造はリストですが、順不同で評価します。
  * 回答が複数項目からなる質問の場合は、JSONオブジェクトで回答
    * 例1: **質問:** *「大豆、春まき小麦、秋まき小麦の限界利益を答えよ。」* --> **回答:** *{"大豆":"81675円,"春小麦":"34896円","秋小麦":"50173円/"}*
    * 例2: **質問:** *「春まき小麦と秋まき小麦の播種時期をそれぞれ答えよ。」* --> **回答:** *{"春まき小麦": "4月中旬", "秋まき小麦": "9月中旬"}*
    * 例3: **質問:** *「人参を100a栽培する際の3月～5月の合計労働時間を答えよ」* --> **回答** *{"3月":"0h", "4月":"53h", "5月":"360h"}*
　* 上記二つの組み合わせを求める質問は、オブジェクト内の値としてリストを持つ形式で回答
    * 例: **質問:** *かぼちゃと人参の収穫時期をそれぞれ答えよ* --> **回答** *{"かぼちゃ": ["8月下旬","9月上旬","9月中旬"], "人参": ["8月上旬","8月中旬","8月下旬"]}*
    * この2段以上複雑なネストを求める質問はありません。
